git init - команда, которая инециирует файл в реппозиторий 

конфигурационные команды, можно конфигурициировать локально или глобально
глобально - значит мы поставим настройки на весь компьютер и все проекты которые мы будем создовать
будут созданы от нашего имени
локально - это значит мы можем в какихто отдельных проектах указать другое имя, если мы указали какие то локальные
данные значит глобальные на этот файл распространяться не будут 
git config --local/global user.name "Bayas"- имя 
git config --local/global user.email ваша почта - почта


Есть три состояния Git:
1 состояние) когда файлы просто созданы 
2 состояние) когда Git следит за какими то файлами, когда такое происходит они попадают в индекс
3 состояние) это когда Git создал контрольные точки, к которым вы сможите вернуться и посмотреть какие были
изменения и посмотреть как работал проект на данной контрольной точке (commit)
для того, что бы посмотреть в каком состаинии проект в консоли надо прописать git status

Что бы перенести файлы в index нужно испоьзовать следующию команду git add -A
